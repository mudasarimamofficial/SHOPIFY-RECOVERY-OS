# Database Schema & Registry

## Core Tables

### `stores`

Stores connected Shopify store configurations and AES-256-GCM encrypted tokens.

- **Columns**:
  - `id` (uuid, PK)
  - `user_id` (uuid, FK `auth.users`)
  - `shop_domain` (text, UNIQUE)
  - `name`, `plan`, `country`, `currency`, `email` (text)
  - `api_version` (text, default '2024-10')
  - `scopes` (text[])
  - `access_token_ciphertext` (text) — **CRITICAL**: Encrypted using AES-256-GCM via `crypto.server.ts`.
  - `status` (text, default 'pending')
  - `auth_method` (text, default 'custom_app_token')
- **RLS**: "own stores" using `((select auth.uid()) = user_id)`.

### `backups`

Records individual backup jobs and their manifests.

- **Columns**:
  - `id` (uuid, PK)
  - `user_id` (uuid, FK)
  - `store_id` (uuid, FK)
  - `status` (text: 'pending', 'running', 'completed', 'failed')
  - `progress` (int)
  - `manifest` (jsonb) — _Note: Contains metadata mapping resource counts to files. Latent bug (BUG-0001) exists where format versions mismatch between backup/restore._
  - `error` (text)
- **RLS**: "own backups".

### `backup_resources`

Granular tracking of individual resources (e.g., Products, Customers) fetched during a backup.

- **Columns**:
  - `id` (uuid, PK)
  - `backup_id` (uuid, FK)
  - `user_id` (uuid, FK)
  - `resource_type` (text)
  - `status` (text)
  - `count` (int)
  - `bytes` (bigint)
  - `recoverability` (text)
- **RLS**: "own backup resources".

### `restore_jobs`

Tracks restore operations moving data from a `backup_id` to a `target_store_id`.

- **Columns**:
  - `id` (uuid, PK)
  - `backup_id` (uuid, FK)
  - `target_store_id` (uuid, FK)
  - `status`, `progress`
  - `plan` (jsonb) — Generated via dependency ordering.
  - `report` (jsonb) — Results of the restore execution.

### `activity_log`

System audit log.

- **Columns**: `id`, `user_id`, `store_id`, `kind`, `title`, `detail`, `meta`.

## Background Queue System

### `jobs`

Enterprise-grade background queue table.

- **Columns**: `id`, `queue_name`, `job_type`, `payload` (jsonb), `status` (queued/running/completed/failed), `priority`, `run_at`, `locked_at`.
- **Indexes**: `idx_jobs_poll` optimized for polling.
- **RLS**: Service-role full access only.

### `job_logs`

Observability for async jobs.

- **Columns**: `id`, `job_id`, `level`, `message`.

## Storage Buckets

- **`recovery_packages`**: 5GB limit, private. Contains `recovery.json` manifests and `*.jsonl.gz` data payloads.
- **RLS**: Users can insert/select/delete their own objects where `auth.uid() = owner`. Service role has full bypass.
