# Function Registry

## `connectShopifyStore` (Server Function)

- **File**: `src/lib/shopify.functions.ts`
- **Purpose**: Verifies a Custom App Admin API token against the Shopify GraphQL endpoint and securely stores it in Supabase using AES-256-GCM.
- **Arguments**: `{ shop_domain: string, access_token: string }`
- **Returns**: `{ store: StoreRecord }`
- **Calls**: `encryptToken()`, Shopify Admin API (`/admin/api/2024-01/graphql.json`)
- **Database Usage**: Upserts to `stores`, Inserts to `activity_log`.
- **Known Bugs**: None.

## `stepBackup` (Server Function)

- **File**: `src/lib/backup.server.ts`
- **Purpose**: Iterates through the resource catalog for a given store, executing Shopify Bulk Operations or REST calls, downloading the data, and pushing compressed JSONL files to Supabase Storage.
- **Arguments**: `{ backupId: string, storeId: string, userId: string }`
- **Returns**: `void`
- **Stages**: Initial -> Fetch -> Compress -> Upload -> Update DB.
- **Errors**: Catch-all for Shopify Rate Limits (429).
- **Dependencies**: `resource-catalog.ts`, `shopify.server.ts`.

## `generateRestorePlan` (Server Function)

- **File**: `src/lib/restore.server.ts`
- **Purpose**: Analyzes a `backup.manifest` and creates an execution plan (dependency tree) mapping how resources should be injected into the target store.
- **Arguments**: `{ backupId: string }`
- **Returns**: `RestorePlan`
- **Known Bugs**: Hardcoded check for `manifest.format === "recovery/1"`, but the backup engine produces `recovery/2`. (See `BUG-0001`).

## `encryptToken` / `decryptToken`

- **File**: `src/lib/crypto.server.ts` (or inline in `shopify.server.ts`)
- **Purpose**: Uses AES-256-GCM with a master `SHOPIFY_TOKEN_ENCRYPTION_KEY` to secure tokens at rest.
- **Dependencies**: Node `crypto` module.
