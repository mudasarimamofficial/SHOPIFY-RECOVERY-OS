## Current Architecture Context

- **Framework**: TanStack Start (React, Vite, Nitro).
- **Backend**: Supabase (Postgres, Auth, Storage).
- **Hosting**: Vercel.
- **Authentication Flow**: We use a **Custom App Authentication Flow**. OAuth is NOT used. The merchant must provide a Shopify Admin API token (`shpat_...`) manually in the UI. **Do not attempt to restore OAuth**.
- **Webhooks**: Subscribed to `app/uninstalled` and GDPR webhooks.
- **Jobs**: Backups and restores are currently synchronous or lightweight async. Background workers/queues might be introduced later.

## Known Production Issues (Historical Context)

- We previously had issues with Vercel SSO (Vercel Protect) blocking callbacks. This is why we rely on the Custom App token flow and use the unprotected production alias for webhooks.
- The Backup engine writes the manifest as `recovery/2` in Supabase Storage (`recovery.json`), but the Restore Engine (`generateRestorePlan`) checks for `recovery/1`. This is a known latent defect. (See `BUG-0001`).

## How to Safely Continue Development

- Do not rewrite history.
- Preserve all existing imports and dependencies unless explicitly removing deprecated features.
- When generating or updating docs, ALWAYS include the YAML frontmatter.
