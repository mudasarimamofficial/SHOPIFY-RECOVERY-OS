# ADR 0001: Move from OAuth to Custom App Authentication

## Problem

The previous application architecture attempted to use a standard Shopify OAuth flow (`/api/auth` -> `/api/auth/callback`). However, the Vercel Edge deployment uses Vercel Protect (SSO), which blocks unauthorized traffic to preview URLs. When Shopify attempted to fire the callback webhook to complete the token exchange, it was blocked by Vercel Protect, resulting in no token being generated and the `stores` table remaining empty for real-world installations.

## Options Considered

1. **Bypass Vercel Protect**: Configure Vercel to allow Shopify IPs to bypass the SSO protection for the `/api/auth/callback` route.
2. **Shopify Token Exchange**: Use Shopify App Bridge Token Exchange. (Rejected because Imam Recovery OS is `embedded = false` and Token Exchange only works for Embedded apps).
3. **Custom App Tokens**: Distribute the app as a "Custom App" within the Shopify Admin, bypassing OAuth entirely and requiring the merchant to provide a statically generated `shpat_...` Admin API token.

## Decision

We elected to use **Option 3 (Custom App Tokens)**. The application is intended to be a highly secure, private recovery OS. Having the merchant explicitly generate and paste an Admin API token fits the security model and bypasses all OAuth callback networking issues.

## Consequences & Tradeoffs

- **Pro**: Completely circumvents Vercel SSO blocking and network callback issues.
- **Pro**: Much simpler authentication flow (no token refresh required, tokens do not expire).
- **Con**: Requires manual user interaction (the merchant must navigate the Shopify Admin, create the app, select scopes, and copy the token).
- **Con**: We cannot automate the full installation flow end-to-end without user interaction.

## Future Migration Path

If we ever list Imam Recovery OS on the Shopify Public App Store, we MUST revert to the OAuth flow and resolve the networking restrictions on the callback URLs.

- `src/routes/_authenticated/connect.tsx` (Restored token input)
- `src/server.ts` (Removed OAuth routes)
- `src/lib/shopify-oauth.server.ts` (Removed OAuth exchange functions)
