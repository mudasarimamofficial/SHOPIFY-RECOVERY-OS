# ADR-0002: Verified Migration Gate and Server-Derived Execution Plans

## Decision

The product must never report a resource as migrated unless the restore handler executed a supported operation, persisted all required ID mappings, and exposed any failure to the job report. The server, not the browser, derives the final execution plan from the backup manifest, selected resources, source domain, target store, and capability catalogue.

## Consequences

- Unsupported, unimplemented, and Shopify-limited resources are explicit `skip` plan items with a reason; they are not silently counted as success.
- A source store cannot be restored into itself.
- Restore jobs finish as `completed_with_failures` when any object failed. They are not eligible for a Recovery Certificate.
- ID mappings are service-role only, scoped to the restore job and owner, and persistence errors abort the affected resource instead of being logged and ignored.
- The application uses Shopify API version `2026-07`; API-version and capability changes must be reviewed each quarter.

## Verification Gate

Production acceptance requires a deployed schema, successful unit/build checks, valid source and destination connections with required scopes, a controlled Store A → Store B migration, property-level comparison, and a signed certificate that records every remaining intentional exclusion or Shopify limitation.
