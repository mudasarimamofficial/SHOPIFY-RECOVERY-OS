# BUG-0001: Restore Engine Manifest Version Mismatch

## Symptoms

When attempting to initiate a restore operation via the Restore Engine, the system will fail or reject the recovery package because the generated restore plan reads `backups.manifest` and strictly expects `format === "recovery/1"`.

## Discovery

Discovered during a code review by Claude Code while reading `restore.server.ts` to document the restore engine.

## Root Cause

The Backup Engine currently writes the recovery package manifest format as `"recovery/2"` to the `recovery.json` file in Supabase Storage. Furthermore, the Backup Engine fails to correctly populate the `backups.manifest` database column during the backup process. However, the Restore Engine explicitly validates the manifest format against `"recovery/1"` and relies on the database column.

## Fix

_(Pending implementation)_
The system needs to either:

1. Update the Backup Engine to write `recovery/1` (if the format hasn't actually changed).
2. Update the Restore Engine to accept `recovery/2` and handle any structural changes introduced in version 2.
3. Ensure the Backup Engine properly inserts/updates the `backups.manifest` JSONB column in the database upon completion.
