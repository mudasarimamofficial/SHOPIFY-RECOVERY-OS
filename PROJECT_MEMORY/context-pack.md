# Universal AI Context Pack

**Project:** Imam Migration OS

## 1. Project Identity

Imam Migration OS is a production-grade Shopify App that extracts store data (Products, Customers, Orders) via the Admin API Bulk Operations, compresses it into JSONL payloads, stores it securely using AES-256-GCM encryption in Supabase, and provides an orchestrated Restore engine to inject that data back into target stores in dependency order.

## 2. Core Architecture

- **Framework**: React / Node.js
- **Database**: Supabase (PostgreSQL)
- **Queue**: Enterprise Postgres Queue (`jobs` and `job_logs`)
- **Authentication**: Shopify Custom App Flow (`access_token` encrypted at rest). OAuth was abandoned due to Vercel SSO blocking.
- **Storage**: Supabase Storage (`recovery_packages` bucket).

## 3. Current State & Blockers

- **Status**: Custom App Authentication is wired up and working.
- **Known Bug [BUG-0001]**: The Backup engine generates `recovery/2` manifests, but the Restore engine (`restore.server.ts`) is hardcoded to validate `recovery/1`. This breaks the restoration pipeline.
